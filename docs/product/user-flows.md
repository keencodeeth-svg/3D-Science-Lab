# 用户流程与信息架构

## 1. 信息架构

### 营销与转化层
- 官网首页
- 方案介绍页
- 试用申请页
- 案例/演示页

### 学生端
- 登录
- 实验大厅
- 实验详情
- 3D 实验页
- 结果与报告
- 历史记录

### 教师端
- 工作台首页
- 实验布置
- 班级管理
- 完成率与分析
- 课堂演示入口

### 学校管理端
- 学校概览
- 账号与组织管理
- 内容权限管理
- 数据报表

## 2. 学生主流程

### 流程目标
让学生从“进入平台”到“完成实验并获得反馈”尽量顺畅。

### 主流程
1. 登录进入实验大厅
2. 按学段/学科筛选实验
3. 查看实验目标、时长、模式
4. 进入引导模式或练习模式
5. 完成器材识别和步骤操作
6. 收到错误提示/阶段反馈
7. 完成实验并生成结果页
8. 查看复盘建议，决定是否再次练习

### 关键体验点
- 进入实验前要先降低理解门槛
- 实验过程中提示不能过密，也不能缺失
- 结果页必须告诉学生“错在哪里、怎么改”

## 3. 教师主流程

### 流程目标
让教师快速完成演示、布置和查看结果。

### 主流程
1. 登录教师工作台
2. 选择实验并预览
3. 选择班级和模式进行布置
4. 课堂上打开演示模式投屏
5. 课后查看完成率、得分和错误节点
6. 决定是否追加练习或二次讲解

### 关键体验点
- 教师第一次使用必须足够简单
- 课堂投屏入口需要足够直接
- 班级结果必须默认看得懂，不要求老师自己分析

## 4. 学校管理者流程

### 流程目标
让学校能看见采购后的使用价值。

### 主流程
1. 管理员开通学校账号
2. 创建年级、班级、教师账号
3. 分配实验内容权限
4. 查看学校使用概览
5. 导出阶段性使用结果或汇报材料

### 关键体验点
- 学校不需要被复杂功能压住
- 管理端重点是“可见成果”和“可控权限”

## 5. 页面优先级建议

### 第一批必须设计
- 官网首页
- 学生实验大厅
- 3D 实验页
- 实验结果页
- 教师工作台首页
- 班级实验结果页

### 第二批再设计
- 学生历史页
- 学校管理页
- 内容管理页
- 数据报表页

## 6. 关键状态设计
- 首次进入状态
- 无实验状态
- 实验进行中状态
- 实验完成状态
- 错误操作状态
- 网络异常/加载失败状态
- 教师无班级数据状态

## 7. 埋点建议

### 学生端埋点
- 进入实验大厅
- 点击实验卡片
- 开始实验
- 完成关键步骤
- 出现错误操作
- 完成实验
- 再次进入同一实验

### 教师端埋点
- 创建布置
- 打开课堂演示
- 查看班级分析
- 二次布置实验

## 8. 页面文案原则
- 用教学语言，不用技术语言
- 尽量用“下一步做什么”提示，而不是抽象说明
- 错误提示先指出问题，再提示修正动作
