# 课程扩容与产品化路线

## 1. 目标
把 `3D Science Lab` 从“少量 3D Demo”升级成“覆盖中小学核心实验的产品级课程平台”。

## 2. 覆盖范围
- 小学：科学
- 初中：物理 / 化学 / 生物
- 高中：物理 / 化学 / 生物

## 3. 扩容原则
- 先高频：优先高频、刚需、容易复用器材的实验
- 先闭环：优先支持教师布置、学生完成、教师复盘的实验
- 先复用：优先建设可复用的交互能力，而不是单个实验的孤立特效
- 先状态化：每个实验都必须明确 `规划中 / 开发中 / 试点可用 / 产品级`

## 4. 产品级定义
一个实验要进入 `产品级`，至少满足：
- 课程锚点完整：主题、单元、知识点清晰
- 交互可判定：关键步骤可操作、可校验
- 考核可输出：能形成得分、错因、反馈
- 教师可复盘：能看到完成情况、步骤回放、错误聚类
- 资产可复用：模型、材质、动画、脚本可沉淀为通用能力

## 5. 建议分期
### Phase 1：课程库底座
- 建立统一 Schema
- 完成学段 / 学科 / 状态筛选
- 建立教师端覆盖看板
- 收拢现有 3D 实验到统一注册表

### Phase 2：小学科学首批实验
- 磁铁的基本性质
- 让小灯泡亮起来
- 水的蒸发与凝结
- 补教师讲义与课堂演示文案

### Phase 3：初中高频实验扩容
- 电学、光学、常见气体制备、显微观察
- 每个学科至少做出 3 个产品级实验
- 沉淀器材库、评分模板、错因模板

### Phase 4：商业化版本
- 登录 / 账号 / 班级任务
- 作业布置、完成率、学校级统计
- 内容生产后台与实验编辑工作流

## 6. 近期研发重点
- 继续把 `frontend/webapp/` 的课程库和教师台做深
- 用 `backend/configs/experiments/` 管理实验生命周期
- 把“新增实验”从改页面，变成“新增配置 + 挂接播放器”
